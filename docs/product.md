# BusDriver Product Overview

## What It Is
BusDriver is a Visual Studio Code extension that helps developers manage Azure Service Bus connections and queue messages directly inside their editor.

## What It Is For
BusDriver is designed to reduce context switching when working with Service Bus by bringing common operational tasks into VS Code, including:

- managing Service Bus namespace connections
- browsing available queues
- viewing queue messages
- sending, moving, and deleting messages

## Target Users
- Developers building or debugging event-driven and messaging-based systems
- Engineers who need quick, repeatable queue/message workflows during local development

## Product Aims
- **Faster troubleshooting**: make queue/message inspection and fixes quick during development.
- **Low-friction workflows**: keep common Service Bus tasks inside VS Code.
- **Safe-by-default operations**: protect sensitive connection data and require explicit destructive actions.
- **Maintainable evolution**: support growth through a testable ports-and-adapters architecture.

## Current Scope
The product currently focuses on core connection and queue-message operations in VS Code, with room to expand into richer diagnostics and management capabilities over time.
